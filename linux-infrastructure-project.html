Master-Slave Architecture

Configured primary DNS server (Master) with automatic zone transfer to secondary DNS server (Slave) for redundancy and load distribution.

Component Server IP Address Function
Master DNS Server 1 192.168.50.10 Primary zone authority
Slave DNS Server 2 192.168.50.20 Automatic zone replication

Zone Configuration

  • Forward Zone: startup.local ‚Üí IP address resolution
  • Reverse Zone IPv4: 50.168.192.in-addr.arpa for PTR records

  • A Records: www, ns1, ns2, dhcp, backup hostnames
  • AAAA Records: IPv6 addressing for all services

üíæ Automated Backup System

Backup Configuration

  • Source Data: Critical configuration files and web content
  • Compression: tar + gzip for efficient storage
  • Transfer Protocol: SCP (Secure Copy Protocol) over SSH
  • Destination: Remote backup server (Server 2)
  • Automation: Cron job scheduled for daily execution
# Backup script example tar -czf /tmp/backup-$(date +%Y%m%d).tar.gz /var/www /etc/bind /etc/dhcp scp /tmp/backup-*.tar.gz backup@192.168.50.20:/backups/

üß™ Integration Testing

Complete System Workflow

Demo Flow:
  1. Client boots and requests IP via DHCP
  2. DHCP server assigns 192.168.50.105 with DNS server info
  3. Client queries DNS for www.startup.local
  4. DNS resolves to 192.168.50.10
  5. Client accesses web page via browser
  6. Administrator blocks client IP with iptables
  7. Client connection denied (firewall enforcement)
  8. Backup system transfers files to remote server
