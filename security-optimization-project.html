üö´ Port Security - MAC Flooding Defense

Understanding MAC Flooding Attacks

In a MAC flooding attack, an attacker sends thousands of frames with different source MAC addresses to a switch. This overflows the CAM (Content Addressable Memory) table, forcing the switch into "hub mode" where it broadcasts all traffic‚Äîallowing the attacker to sniff packets destined for other devices.

Attack Scenario: Without port security, an attacker connected to a single switchport could flood the CAM table with 100,000+ fake MAC addresses in seconds, causing the switch to broadcast all traffic and exposing sensitive data.

Implementation: All access ports configured with maximum 2 MAC addresses per port using restrict mode, providing security with availability and logging.

‚ö° EtherChannel with LACP

Link Aggregation Benefits

EtherChannel combines multiple physical links into a single logical link, providing increased bandwidth and redundancy. Using LACP (Link Aggregation Control Protocol), the system can automatically detect link failures and redistribute traffic across remaining links.

  • Bandwidth Aggregation: 2 x 1 Gbps = 2 Gbps combined throughput
  • Automatic Failover: If one link fails, traffic instantly moves to the other
  • Load Balancing: Traffic distributed across all active links
  • No Spanning Tree Blocking: All links forward traffic simultaneously

Implementation: New York Office

Configured EtherChannel between NY router and switch using two GigabitEthernet interfaces with LACP for dynamic negotiation, achieving 2 Gbps aggregate bandwidth with automatic failover.

Understanding Spanning Tree Protocol

STP prevents Layer 2 loops by creating a loop-free logical topology. Without STP, broadcast frames would circulate indefinitely, causing a broadcast storm that brings down the network within seconds.

Protocol Convergence Time Per-VLAN
STP (802.1D) 30-50 seconds ‚ùå Single instance

PortFast & BPDU Guard

  • PortFast: Allows access ports to bypass STP listening/learning states, bringing up end devices instantly
  • BPDU Guard: Protects against accidental switch connections that could disrupt the STP topology by immediately err-disabling ports that receive BPDUs
