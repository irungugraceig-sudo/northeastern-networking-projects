üìã Project Overview

Designed and implemented a static routing network for a robotics company expanding its infrastructure using a Class C network address (195.202.88.0/24). Applied Variable Length Subnet Masking (VLSM) to efficiently allocate IP addresses across six departments with varying host requirements.

üéØ Challenge: Efficiently subnet a single Class C network (256 total addresses) to accommodate 6 departments ranging from 16 to 34 hosts each, minimizing address waste while maintaining scalability.

üßÆ VLSM Calculation Process

Step 1: Determine Required Subnet Sizes

Sales (34 hosts):
Need: 34 + 2 (network + broadcast) = 36 addresses
Next power of 2: 2^6 = 64 addresses
Result: /26 subnet (62 usable)
Customer Support (30 hosts):
Need: 30 + 2 = 32 addresses
Exactly: 2^5 = 32 addresses
Result: /27 subnet (30 usable)
Other Departments (16-26 hosts):
Need: 26 + 2 = 28 addresses max
Next power of 2: 2^5 = 32 addresses
Result: /27 subnet (30 usable) for each

üéØ Key Concepts Demonstrated

Class C Network Analysis

  • Network Bits: First 24 bits (195.202.88) represent the network portion
  • Host Bits: Last 8 bits available for host addressing
  • Maximum Subnets: 2^6 = 64 possible subnets (leaving 2 bits for hosts)
  • Minimum Subnet Size: /30 (2 usable hosts) to maintain 4 addresses per subnet

VLSM Benefits

  • Efficiency: Allocates appropriate subnet sizes based on actual requirements
  • Waste Reduction: Sales gets 62 IPs (not 126), others get 30 IPs (not 62)
  • Scalability: Each department has room for growth within assigned range
  • Address Conservation: Unused addresses (224-255) available for future expansion

Subnet Mask Calculation

/26 Subnet:
Binary: 11111111.11111111.11111111.11000000
Decimal: 255.255.255.192

/27 Subnet:
Binary: 11111111.11111111.11111111.11100000
Decimal: 255.255.255.224
Host bits: 5 ‚Üí 2^5 = 32 total, 30 usable

Static Routing Configuration

Each department connected via static routes configured on edge routers. No dynamic routing protocols used, ensuring:

  • Predictable routing paths
  • Lower overhead on router resources
  • Enhanced security through explicit route control
  • Simplified troubleshooting for small-scale network
